# 管理者ページ（コート予約の編集）セットアップ／カットオーバー手順

コート予約を管理者がブラウザから修正できる仕組みです。
**正データ＝公開フォルダの外のJSON**、**公開ページ（月別HTML）＝JSONから自動生成**、という構成です。

---

## 1. 構成の全体像

```
kasukou2025.stars.ne.jp/                ← public_html の1つ上（Web非公開）
├── admin_config.php                    ← 管理者ID/パスワード（FTPで設置）
├── reservation_data/<年>/<月>-<場所>.json  ← 正データ（PHPが自動生成）
└── public_html/                        ← Web公開
    ├── admin/                          ← 管理者ページ（ログイン・編集・各種ツール）
    └── coatbokking/NEWFORMAT/<年>/coatbooking<月>-<場所>.html  ← JSONから生成される公開ページ
```

- 予約データ(JSON)と設定は**公開フォルダの外**にあるため、Webから直接見えず、デプロイ／FTP同期でも触られません。
- 公開ページHTMLは**アプリ（管理者ページ）が生成・管理**します。`deploy.yml` で `coatbokking/NEWFORMAT/**` を**デプロイ除外**しているので、コードのデプロイで予約ページが上書きされることはありません。

---

## 2. 初回セットアップ（FTPで1回だけ）

`public_html` の**1つ上**の階層（`cancel_config.php` と同じ場所）に **`admin_config.php`** を置きます。
テンプレート `public_html/cancel_config.sample.php` ではなく、リポジトリ直下の **`admin_config.sample.php`** をコピーして使ってください。

```php
<?php
return [
    'admin_user' => 'お好きなID',
    // 推奨: ハッシュ（php -r "echo password_hash('パスワード', PASSWORD_DEFAULT);" で生成）
    // 'admin_pass_hash' => '$2y$...',
    // 暫定: 平文（PHPファイルなのでWebには出ません）
    'admin_pass' => '実際のパスワード',
];
```

- 置けない場合は `public_html/admin/admin_config.php` でも動きます（その場合はGitHub同期に乗る点に注意）。
- `reservation_data/` フォルダは手動作成不要（最初の保存時にPHPが自動作成）。

---

## 3. カットオーバー（本番を新方式へ切替）

デプロイ（PRマージ）後に、ブラウザで一度だけ行います。

1. **ログイン**：`https://kasukou2025.stars.ne.jp/admin/login.php`
2. **「予約データの移行（既存HTML→JSON）」を実行**
   - 本番の現在の月別HTMLを読み取り、正データ(JSON)を作成します（最新の○×を取り込む）。
   - 既存HTMLは書き換えません。
3. （任意）**「全月いっせい再生成（新見た目に統一）」を実行**
   - 全公開ページを生成版（新しい見た目・ナビ付き）に統一します。
   - 予約データ(○×)は変わりません。見た目だけ統一されます。
   - ここで初めて公開ページが新見た目に変わります。様子を見ながら年単位でも実行できます。

> 順番が重要：必ず **「移行」→「再生成」** の順で。先に再生成すると取り込み前で対象がありません。

---

## 4. 日常の使い方

- **編集**：管理トップ → 対象月の「編集」→ 各マスをクリックで 〇⇄×、備考入力 → 「保存」
  - 保存すると JSON が更新され、その月の公開ページHTMLが自動再生成されます。
- **日付の追加・削除**：編集画面で「日付を追加」／各行の「削除」。曜日は日付から自動。
- **新しい月を作成**：管理トップ →「新しい月を作成」→ 年月・場所を指定 → 編集画面で日付を追加。

---

## 5. パスワードの変更

`admin_config.php` の `admin_pass`（または `admin_pass_hash`）を書き換えて保存するだけ。
保存した瞬間から有効です（デプロイ不要）。

---

## 6. よくある確認

- ログインできない／設定未設置 → `admin_config.php` の場所・名前・中身を確認。
- 公開ページが古い見た目のまま → その月をまだ「編集保存」または「再生成」していない（未編集の月は元の手書きHTMLのまま表示されます）。
- 中止連絡（雨・熱中症など）の機能はこれまで通り動きます（生成HTMLにも中止表示スクリプトが入っています）。
